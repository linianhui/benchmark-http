# 1 HTTP Benchmark 环境准备

# 1.1 准备2个HTTP服务器
```sh
docker-compose up --detach --build
```

| server port | note                         |
| ----------- | ---------------------------- |
| 60001       | http/1.1                     |
| 60002       | http/2 http2-prior-knowledge |


验证是否成功启动
```sh
curl --http1.1 -v http://192.168.2.201:60001

curl --http2-prior-knowledge -v http://192.168.2.201:60002
```

# 1.2 准备1个HTTP客户端

```sh
docker build --tag http-client ./client/
```

# 术语

1. http/1.0 : http/1.0
2. http/1.1 : 基于tcp，默认支持长连接
3. h2       : 基于tls，支持多路复用.
4. h2c : 基于tcp, 明文版的h2.
5. http2-prior-knowledge : 基于tcp，客户端不经过http的协议升级协商，直接发起2的请求.
